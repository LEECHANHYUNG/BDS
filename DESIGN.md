# DESIGN.md — 시각 기준

이 문서는 BDS(React 디자인 시스템)의 **시각·표현 기준**이다. 컴포넌트, 토큰, 스토리, 문서 사이트, 시각 산출물은 모두 이 기준을 따른다. 두 도구 모두 산출물을 만들기 전에 이 문서를 참조한다.

> 이 문서는 사람이 관리하는 기준이다. 값이 비어 있는 항목(`_(미정)_`)은 토큰·컴포넌트 설계가 진행되며 사람이 확정한다. AI는 비어 있는 값을 임의로 채워 단정하지 않는다 — 필요하면 조사/계획 단계에서 제안하고 사람의 승인을 받는다.

## 디자인 원칙

제품 경험의 신뢰성과 명료함을 해치지 않는 것이 최우선이다.

1. **신뢰 우선** — 모호한 상태·과장된 시각 효과를 피한다. 사용자가 중요한 정보를 다루는 화면이라는 전제.
2. **명료함** — 핵심 정보와 상태는 한눈에 읽혀야 한다. 장식보다 가독성.
3. **일관성** — 같은 의미는 항상 같은 색·간격·컴포넌트로 표현한다. 토큰을 우회한 하드코딩 금지.
4. **접근성 기본 탑재** — 대비·키보드·스크린리더 지원은 옵션이 아니라 기본값.

## 색상 (토큰)

> 실제 값은 토큰 패키지에서 단일 출처로 관리한다. 여기서는 **의미(semantic) 역할**만 규정한다.

| 역할                   | 의미             | 값       |
| ---------------------- | ---------------- | -------- |
| `primary`              | 주요 액션·브랜드 | _(미정)_ |
| `success` / `positive` | 성공·긍정 상태   | _(미정)_ |
| `danger` / `negative`  | 오류·위험 상태   | _(미정)_ |
| `warning`              | 주의·경고        | _(미정)_ |
| `neutral`              | 본문·배경·테두리 | _(미정)_ |

- 의미 토큰만 사용하고, 컴포넌트에서 raw 색상값을 직접 쓰지 않는다.

## 타이포그래피 / 숫자

- 수량·지표는 **고정폭(tabular) 숫자**로 정렬해 자릿수가 흔들리지 않게 한다.
- 천 단위 구분·소수 자릿수 포맷은 컴포넌트가 아니라 공용 포맷 유틸로 일관 처리한다.
- 본문/숫자/라벨 스케일은 토큰으로 관리한다. _(스케일 값 미정)_

## 간격 / 레이아웃 / 모서리

- 간격·라운드·그림자(elevation)는 토큰 스케일을 따른다. _(값 미정)_
- 임의 픽셀 값 하드코딩 금지 — 토큰에 없으면 토큰을 먼저 추가한다.

## 접근성 기준

- 텍스트 대비: WCAG **AA 이상**(본문 4.5:1, 큰 텍스트 3:1). 가능하면 핵심 정보는 AAA를 지향.
- **색만으로 의미 전달 금지** — 성공/오류 등은 색 + 아이콘/부호/라벨을 함께 쓴다(색각 이상 고려).
- 모든 인터랙티브 요소는 키보드 접근·가시적 포커스 링·적절한 ARIA를 갖춘다.

## 금지 사항

- 실제 브랜드 **로고·상표** 무단 사용.
- **효능·성과 과장 표현** 및 오해 소지 시각화.
- 데이터 없는 **추정 수치·가짜 그래프**를 실제처럼 보이게 하는 시각화.
- 토큰을 우회한 raw 색상/간격 하드코딩.
