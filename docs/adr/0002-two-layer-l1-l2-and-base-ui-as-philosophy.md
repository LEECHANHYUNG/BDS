# 2층 구조(L1/L2)와 base-ui는 의존이 아닌 인터페이스 철학 차용

이 시스템은 두 층으로 나뉜다: **L1(범용 프리미티브)** — Button/Input/Dialog 등 무도메인 컴포넌트, **L2(증권 도메인 컴포넌트)** — 실시간 시세·호가창·가격입력 등. L1은 L2를 만들기 위한 토대일 뿐 차별점이 아니며, **L2가 이 시스템의 본체**다. 이렇게 명시적으로 구분하지 않으면 "예쁜 범용 DS"를 만들고 끝나 C(금융 도메인)를 전혀 증명하지 못하는 흔한 실패에 빠진다 — 금융 도메인은 base-ui/MUI에 _없는_ L2에서만 증명된다.

base-ui는 **런타임 의존으로 가져다 쓰지 않고, 그 인터페이스 철학(prop 설계·composition·controlled/uncontrolled)만 차용**한다. base-ui를 그대로 의존하면 결과물이 "base-ui 위에 스타일 입힌 또 하나의 범용 DS"가 되어 차별점이 흐려지기 때문이다. (단, 실제 런타임 의존 여부 자체는 조사 단계에서 재검토 가능하다 — 여기서 고정하는 것은 "철학을 기준으로 삼는다"는 방향이다.)

## Consequences

- L2 컴포넌트 목록은 감으로 고르지 않고 **주식주문 플로우(ADR-0003)에서 역산**한다.
- 금융 도메인 근거는 두 소스로 채운다: ①접근성 규제(KWCAG 등 — 정확한 문서는 조사 단계에서 특정), ②실제 금융 앱 관습(웹 조사).
