# 쇼케이스 플로우 = 주식주문, 데이터는 mock 실시간

데모 앱의 쇼케이스 플로우로 **주식주문**(종목 조회 → 가격/수량 입력 → 주문 확인 → 체결)을 택한다. 증권 도메인은 뱅킹보다 도메인 특수성이 진해(실시간 수치, 손익 색상 관습, 시세 신뢰 상태) C(금융 도메인) 증명에 유리하다. 이 플로우가 **L2 컴포넌트 목록을 역산하는 기준점**이다: 종목 검색/선택, 실시간 시세, 호가창, 가격 입력(stepper), 수량 입력, 주문 요약/확인, 손익 표시.

데이터는 **mock 실시간**으로 한다 — 타이머로 호가가 흔들리고 "지연/끊김" 신뢰 상태를 일부러 토글할 수 있는 가짜 데이터다. 디자인 시스템 데모의 목적은 데이터의 진위가 아니라 **컴포넌트가 상태(실시간성·신뢰 상태)를 어떻게 표현하는가**이기 때문이다.

## Considered Options

- **실 증권 API 연동**: 웹소켓·인증·연동에 시간이 쏠려 C/A와 무관한 B(인프라)의 늪이 됨. 기각.
- **정적 더미 수치**: 주식주문의 본질(실시간성·신뢰 상태)을 증명하지 못해 "예쁜 폼"에 그침. 기각.
