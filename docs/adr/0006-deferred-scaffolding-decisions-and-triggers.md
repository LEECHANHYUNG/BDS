# 골격 단계에서 의식적으로 보류한 것들과 재평가 트리거

골격(ADR-0004·0005)만 확정하고, 아래 요소들은 **이번에 정하지 않기로 의식적으로 결정**한다. "아직 안 본 것"이 아니라 "지금 정하면 근거 없는 박제가 되거나 B 과투자가 되므로 미룬 것"이다. 각 항목의 **재평가 트리거**를 함께 박아, 다음 세션이 "이거 정해야 하나?"로 흔들리지 않게 한다. (보류 판단의 의존 순서 근거: `docs/research/monorepo-foundation.md`.)

| 보류 항목                                                                          | 왜 지금 보류                                                                                             | 재평가 트리거                                                                                                                                                             |
| ---------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **공유 tsconfig 패키지** (`@bds/tsconfig`)                                         | 공유 대상 패키지가 0개 — 지금 만들면 빈 부채                                                             | 패키지가 2~3개로 늘어 tsconfig 중복이 생길 때 추출                                                                                                                        |
| **번들러** (tsdown / tsup) + **번들 포맷** (ESM-only vs dual) + **`exports` 설계** | 컴포넌트 0개, 소비처 환경(RSC·Next 버전·CJS 필요 여부) 미확정. tsup은 유지보수 중단, tsdown은 0.x 불안정 | 첫 컴포넌트 패키지를 빌드·publish하는 시점에 소비처 확인 후 결정 → [ADR-0009](0009-design-tokens-static-css-button-slice.md)로 Vite library mode + 정적 CSS exports 결정  |
| **원격 캐시 / CI** (Turborepo Remote Cache, GitHub Actions, `changesets/action`)   | 보안·인프라 정책 미확정. 로컬 캐시로 충분                                                                | CI 파이프라인을 도입하는 시점 (빌드 오케스트레이터는 이미 turbo로 확정)                                                                                                   |
| **토큰 파이프라인** (DTCG → Style Dictionary → CSS/TS)                             | 토큰 값 자체가 `DESIGN.md`에서 미정. 골격이 아니라 토큰 작업                                             | `@bds/tokens` 패키지의 토큰 값을 채우는 작업 단계 → [ADR-0009](0009-design-tokens-static-css-button-slice.md)로 SD 5.4.4 + 3층 토큰 최소 슬라이스 결정                    |
| **Storybook 10 / Chromatic**                                                       | 컴포넌트 0개라 보여줄 스토리가 없음                                                                      | 첫 L1 컴포넌트가 생기는 시점 (a11y 대비 검증도 여기에 연동)                                                                                                               |
| **린트 a11y 보강** (ESLint `jsx-a11y` 보조, axe-core)                              | ADR-0004 Biome 결정의 조건부 후속                                                                        | 첫 컴포넌트 추가 시 Biome a11y 룰 부족분 실측 후 → [ADR-0009](0009-design-tokens-static-css-button-slice.md)에서 oxlint jsx-a11y 유지, axe 계열은 Storybook 시점으로 보류 |

## Consequences

- 위 항목들은 ADR이 아직 없는 게 정상이다 — 재평가 트리거가 발동하는 시점에 각각 별도 ADR로 결정한다.
- 특히 **번들 포맷·`exports`**는 ADR-0005의 "L1/L2 서브패스 노출 방식"과 묶여 있어 같은 시점(첫 컴포넌트 빌드)에 함께 결정한다.
- 이 "explicit no" 목록은 포폴 관점에서도 의미가 있다 — 인프라를 무지성으로 다 깔지 않고 **필요 시점에 맞춰 도입한다는 판단**을 보여주는 기록이다(ADR-0001 C>A>B 일관).
