# BDS — 금융(증권) 디자인 시스템 도메인

BDS는 한국 금융(증권) 도메인에 특화된 공개 npm 디자인 시스템이다. 범용 프리미티브(L1) 위에 증권 도메인 컴포넌트(L2)를 얹으며, **주식주문 플로우**를 쇼케이스로 삼아 L2를 역산한다. 이 문서는 그 도메인의 **언어(용어집)** 만 담는다 — 시각 기준은 [`DESIGN.md`](DESIGN.md), 결정 근거는 [`docs/adr/`](docs/adr/)에 있다.

## Language

### 계층

**프리미티브 (L1)**:
base-ui의 인터페이스 철학(prop·composition·controlled/uncontrolled)을 따른 범용 무도메인 컴포넌트(Button, Input, Dialog 등). 금융 컴포넌트의 토대일 뿐 차별점은 아니다.
_Avoid_: base 컴포넌트, 공용 컴포넌트

**도메인 컴포넌트 (L2)**:
증권 도메인 지식이 녹아든 컴포넌트(실시간 시세, 호가창, 가격 입력 등). 이 시스템의 본체.
_Avoid_: 금융 위젯, 특수 컴포넌트

### 주식주문 도메인

**종목**:
거래 대상 자산. 종목코드와 종목명으로 식별한다.
_Avoid_: 주식, 상품, 아이템, symbol(코드 자체를 가리킬 때만 종목코드)

**시세**:
종목의 현재 가격과 등락 정보(현재가·등락폭·등락률). 실시간으로 갱신된다.
_Avoid_: 가격(price는 호가/주문가와 구분), 시가(개장가만 가리킴)

**호가**:
매수·매도 주문이 대기 중인 가격과 수량의 목록. 호가창(order book)으로 표시한다.
_Avoid_: 매물대, 오더북(영문 표기 지양)

**호가 단위**:
주문 가격이 움직일 수 있는 최소 간격(tick). 가격대별로 다르다.
_Avoid_: 틱, 가격 스텝

**주문**:
사용자가 제출하는 매수/매도 의사. 종목·가격·수량·주문유형으로 구성된다.
_Avoid_: 거래, 트랜잭션, 매매(체결과 혼동)

**체결**:
주문이 실제로 성사된 결과. 주문과 구분된다(주문≠체결).
_Avoid_: 거래 완료, 성사

**손익**:
보유·체결 결과의 이익 또는 손실. 부호·색·퍼센트로 표시한다.
_Avoid_: 수익(이익만 가리킴), 평가금액(손익과 다름), PnL

### 신뢰 상태 (금융 UI 핵심)

**증감 색상**:
상승/하락을 나타내는 색 관습. 한국 시장은 상승=빨강/하락=파랑이 통상이나 지역마다 반대다. 색만으로 전달하지 않고 항상 부호·아이콘·라벨을 함께 쓴다(색각 이상 고려). 실제 값은 토큰으로 확정한다([`DESIGN.md`](DESIGN.md)).
_Avoid_: 등락 색, 상승하락 컬러

**시세 신뢰 상태**:
표시 중인 수치가 얼마나 믿을 수 있는지를 나타내는 메타 상태(실시간 / 지연 / 끊김). 금융 신뢰 문제의 핵심이며, 데모에서는 mock으로 토글한다.
_Avoid_: 연결 상태, 데이터 상태

**비가역 경고**:
주문 제출처럼 되돌릴 수 없는 액션 직전에 사용자에게 결과(예상 체결금액·수수료)를 명확히 고지하는 표시.
_Avoid_: 확인 모달, 경고 팝업
